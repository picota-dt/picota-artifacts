# Picota Artifacts

This repository contains the **reproducibility artifacts** for Picota: DSML models, public datasets (when permitted), and supporting material used in the case studies reported in the paper.

- Software implementation: https://github.com/picota-dt/picota  
- Artifacts repository (this): https://github.com/picota-dt/picota-artifacts  
- Recommended version/tag for reproducibility: **1.3.11**

---

## Repository layout

Artifacts are organized by case study under `study_cases/`.  
Each case study folder contains:

- `model.tara` — the **Picota DSML model** used in Quassar/Picota for that case.
- `README.md` — case-specific notes.
- Public datasets.
- `report.pdf` — evaluation report generated by Picota (see **Expected outputs**).

Current relevant paths:

- `study_cases/households-es/`
  - `model.tara`
  - `dataset.tsv` — public dataset in the expected schema for the household case study
  - `report.pdf` — generated evaluation report

> Note: this repository does **not** provide explicit schema files. The expected data structure is defined by the corresponding `model.tara` and the Picota/Quassar ingestion conventions.

## Requirements

Picota is intended to run in a **GPU-enabled runtime**.

### Stack (reference)
- Java: **21**
- Python: **3.12**
- PyTorch: **2.7.0**
- Docker Engine (recommended) + NVIDIA Container Toolkit (for GPU access)

This repository contains **artifacts and datasets**. The Picota runtime and build tooling live in the software repository.

## Quickstart (Households ES case study)

This is the **fully reproducible** case study using the public dataset shipped in this repository.

### 1) Clone repositories and checkout the recommended tag

```bash
git clone https://github.com/picota-dt/picota
git clone https://github.com/picota-dt/picota-artifacts

cd picota
git checkout 1.3.11

cd ../picota-artifacts
git checkout 1.3.11
```

### 2) Build the Picota Docker image (GPU runtime)

From the **Picota software** repository:

```bash
cd ../picota
./docker-build
```

> If your environment requires CUDA-specific configuration, refer to the Picota repository documentation.

### 3) Deploy the model via Quassar and run Picota

1. Go to **https://quassar.io**
2. Open the case study model:
   - `picota-artifacts/study_cases/households-es/model.tara`
3. Deploy it to Picota.
4. Provide the dataset:
   - `picota-artifacts/study_cases/households-es/dataset.tsv`

Picota will train and evaluate the estimator(s), producing:
- a **PDF evaluation report** including **MAE/MSE** per predicted variable and **feature influence** information
- deployable DT inference services (endpoints) and runtime quality indicators (including MR-related indicators when defined in `model.tara`)

## Dataset notes

### TSV conventions
- All `.tsv` files **include a header row**.
- Any row containing **blank/empty values** is ignored (the row is discarded).

### Households ES dataset (`study_cases/households-es/dataset.tsv`)
- Format: TSV (tab-separated values)
- Intended to match the variables and types used by `study_cases/households-es/model.tara`.

Since there are no separate schema files in this repository, the recommended way to understand the expected fields is:
1) open `model.tara` and identify the subject variables and targets, and  
2) inspect the dataset header/columns to confirm naming consistency.

> Tip: If you want to make this repository even easier to reuse, add a short `study_cases/households-es/README.md` listing column names, types, units, and missing-value encoding.

## Expected outputs

Picota generates an evaluation report in **PDF** that includes:
- MAE/MSE per predicted variable
- feature influence (most influential inputs per prediction)

The report is stored inside each case study folder as:

- `study_cases/<case-name>/report.pdf`

Additionally, Picota deploys inference services and exposes runtime quality indicators (including MR-related indicators when present in the model).

## Troubleshooting

### GPU not detected in Docker
- Ensure the NVIDIA driver is installed on the host.
- Install NVIDIA Container Toolkit.
- Verify with:

```bash
nvidia-smi
docker run --rm --gpus all nvidia/cuda:12.4.1-base-ubuntu22.04 nvidia-smi
```

### Dataset ingestion issues
- Ensure `.tsv` files use tabs (not spaces/commas).
- Ensure decimals use `.` consistently.
- Ensure categorical values match enumerations in `model.tara`.
